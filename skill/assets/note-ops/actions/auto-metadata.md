根据以下指示，智能补全或更新当前 Markdown 笔记的 YAML Frontmatter（元数据）：

### 任务说明
1. **分析笔记内容**：仔细阅读正文，判断它的题材、当前状态、核心主题。
2. **生成 Frontmatter 字段**：
   - `type`：从以下候选值中选择一个最合适的：`公众号文章`, `生图实验`, `项目复盘`, `AI资料`, `工具评测`, `流程方法`, `入口`
   - `status`：候选值：`草稿`, `进行中`, `已完成`, `待整理`
   - `topic`：提炼一个简短的主题名称（如：`AI生图`、`AI编程`、`知识管理`、`科学上网`、`多智能体` 等）
   - `created`：保留原有日期，若原先没有，则设置为今天的日期（使用你运行任务当天的 `YYYY-MM-DD`）
   - `tags`：生成 2-4 个相关的标签列表（以 `- 标签名` 格式表示，不要带 `#` 号，包含 `type` 的值作为一个标签）
3. **元数据合并与更新**：
   - 如果原笔记头部**已经存在** `---` 包裹的 YAML 元数据，请保留原有的其他自定义字段，仅新增或覆盖 `type`、`status`、`topic`、`created`、`tags` 这几个核心字段。
   - 如果原笔记头部**不存在** `---` 元数据，请在最顶部新建它。
4. **输出限制（极其重要）**：
   - 只输出修改后的**完整笔记原文**，以包含新 Frontmatter 的 `---` 开头。
   - **绝对不要**在最外层包裹 ` ```markdown ` 或 ` ``` ` 代码块。
   - **绝对不要**输出任何解释性话语、前言或后缀。
   - 保持笔记正文（Frontmatter 之外的部分）完全不变，不要修改、润色或删减正文内容。
